# Next Product Landing UI

Build landing pages like product surfaces, not marketing fog: clear promise, useful first viewport, native controls, crisp contrast, and warm editorial support sections.

Do:

- Make the first viewport immediately useful.
- Put the product promise in plain language.
- Use dark editorial bands for command and proof.
- Use warm paper sections for explanation and examples.
- Keep buttons rectangular with 6px radius.
- Verify mobile and desktop layouts.

Don't:

- Use oversized decorative hero cards.
- Use abstract gradients as the main visual idea.
- Add nested cards or feature-card clutter.
- Hide the actual product workflow below vague copy.

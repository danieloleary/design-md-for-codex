Why DESIGN.md works

Your taste belongs in the repo.

Prompts disappear. Rules remain. Codex knows where to look.

Rules travel with the code

Colors, type, spacing, components, accessibility, voice, and no-go zones stay in one file.

Prompts get smaller

Stop retyping the same visual preferences every time Codex touches a screen.

Reviews get sharper

Give feedback once. Put it in the file. Let future work inherit it.

UI stops drifting

New pages start from the same source instead of a fresh visual guess.

The habit

Before: Papyrus soup. After: governed workbench.

Without it

Codex can ship loud default UI: gradients, blobs, pill buttons, huge radii, and a new visual system every screen.

With it

Codex reads DESIGN.md, applies the tokens, turns noise into a workbench, and checks desktop and mobile.

Why DESIGN.md rocks

Taste stops living in chat.

Put the product rules where Codex can find them every time.

Taste lives in the repo

Colors, type, spacing, components, accessibility, voice, and no-go zones travel with the code.

Prompts get shorter

Stop retyping the same visual preferences every time you ask for a screen.

Reviews get less repetitive

Codex has a source of truth for the feedback you keep giving.

UI stops drifting

New pages start from the same tokens and rules instead of a fresh vibe roll.

See the habit

Before: decent UI. After: UI with memory.

Without this skill

Codex can build something usable, but it may invent new spacing, accent colors, rounded cards, and shadows screen by screen.

With this skill

Codex reads DESIGN.md, applies tokens and component rules, then checks the rendered UI at desktop and mobile widths.

Test it yourself

Two-minute proof loop.

This repo includes a QA fixture and smoke script so the showcase can be tested like a real product.

  1. Copy the install prompt into Codex.
  2. Restart Codex.
  3. Add DESIGN.md to a repo.
  4. Run $design-system Make this page follow DESIGN.md.
  5. Confirm Codex read or applied the rules in its response, diff, or rendered UI.

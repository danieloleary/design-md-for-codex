Why DESIGN.md rocks

Prompts disappear. Rules remain.

DESIGN.md puts taste where agents can find it: colors, type, spacing, components, accessibility, voice, and no-go zones.

Taste lives in the repo

Your design rules travel with the code, not with a forgotten chat thread.

Prompts get shorter

Stop re-explaining buttons, spacing, colors, shadows, and vibe every time.

Reviews get less repetitive

Give feedback once, put it in the file, and let future UI work inherit it.

UI stops drifting

Codex starts from a source of truth instead of a fresh visual guess.

Before and after

Before: Papyrus soup. After: governed workbench.

The skill does not promise magic. It creates a habit: read the file, apply the rules, check the screen.

Test it yourself

Five steps. No mythology.

  1. Copy the install prompt into Codex.
  2. Restart Codex so the skill loads.
  3. Add DESIGN.md to your repo.
  4. Run $design-system Make this page follow DESIGN.md.
  5. Confirm Codex read the file and used the rules in the diff or rendered UI.
